# REST API v1

AION Agent exposes a suite of RESTful APIs under the `/v1` namespace designed to allow the integration of the agent into any application (web, mobile, backend-to-backend).

---

## Authentication

The endpoints under `/v1/*` support two distinct authentication modes depending on the resource type:

### 1. API Key (Programmatic Server-to-Server)
Required for managing conversations (`/v1/conversations/*`) and their related files (`/v1/conversations/*/files`).
- **Header**:
  ```http
  X-API-Key: aion_..._...
  ```
  *(or via `Authorization: Bearer aion_..._...`)*
- API Keys have granular scopes (e.g. `conversations:read`, `conversations:write`, `files:read`, `files:write`). In case of insufficient scope, the API will return a `403 Forbidden` error.

### 2. Chat Authentication (User and BFF)
Required for chat/streaming interactions (`/v1/chat/*`), MCP integrations (`/v1/integrations/*`), Query Memory (`/v1/query-memory/*`) and Navigation Memory (`/v1/navigation-memory/*`). 
Accepts one of the following authorization methods:
- **Bearer Token Chat**: Opaque HMAC token generated by `/auth/login` (typically used by the Chat UI).
  ```http
  Authorization: Bearer <token_hmac>
  ```
  *(for SSE endpoints like `EventSource`, which do not support custom headers, it is possible to pass the token in the query string: `?access_token=<token_hmac>`)*
- **BFF Internal Secret**: Server-to-server header to bypass login in trusted scenarios.
  ```http
  X-AION-Chat-Ui-Secret: <shared_secret>
  ```
- **API Key**: An API Key with `chat` or `admin` scope.
- **Open Mode**: If `AION_CHAT_PASSWORD_AUTH=0` in the backend, read-only operations are no-op (anonymous identity allowed). Write operations will still require a verified identity.

## Management of `profile` and `project` Parameters

The customization of the agent's behavior and the isolation of data are based on two fundamental concepts:

### Profile (`profile`)
Identifies the agent's configuration (LLM model, system prompt, set of enabled MCP tools).
- **Conversation Creation** (`POST /v1/conversations`): Sent in the JSON body as `"profile"` or `"profile_slug"`. Permanently associates the session with that profile.
- **Chat and Streaming** (`POST /v1/chat/stream`, `POST /v1/chat/prepare`): Sent in the JSON body as `"profile"`, `"profile_slug"` or `"profile_name"`. If omitted, the default value is `aion_std`.
- **Query Memory** (`GET/POST /v1/query-memory/projects`): Passed as `profile` query parameter to filter or associate projects.

### Project (`project`)
Identifies the SQL QueryMemory slot to share and catalog pre-validated queries or user preferences.
- **Chat and Streaming** (`POST /v1/chat/stream`): Sent in the JSON body as `"sql_query_project"` (e.g. `"default"`, `"vendite"`, `"tecnico"`).
- **Query Memory API** (`/v1/query-memory/*`): Passed as `project` query parameter (e.g. `?project=vendite`).
- **Navigation Memory API** (`/v1/navigation-memory/*`): Passed as `project` query parameter or inside the JSON body as `"project"` for upserts.

#### SQL Project Resolution Flow
When a message is sent in the chat, the active project is determined with the following priority:
1. **Explicit request**: The value of `"sql_query_project"` present in the body of `POST /v1/chat/stream`.
2. **Conversation metadata**: The value stored in the `Conversation` table (under `metadata_json -> sql_query_project`) generated at the previous turn or at creation.
3. **System fallback**: The value of the environment variable `AION_SQL_QM_DEFAULT_PROJECT` set on the server (default: `"default"`).

## Conversations Management

### Create Conversation
`POST /v1/conversations` (Requires `conversations:write`)

**Request Body:**
```json
{
  "profile": "aion_std",
  "user_id": "jane.doe",
  "title": "Infrastructure Analysis Q2",
  "metadata": {"source": "web-portal"},
  "tags": ["audit", "q2"]
}
```

**Response (201 Created):**
```json
{
  "id": "01907a5e-5df3-7b9a-9f44-3c1d4d2e7f3a",
  "tenant_id": "default",
  "user_id": "jane.doe",
  "profile_slug": "aion_std",
  "title": "Infrastructure Analysis Q2",
  "message_count": 0,
  "metadata": {"source": "web-portal"},
  "tags": ["audit", "q2"]
}
```

### List Conversations
`GET /v1/conversations` (Requires `conversations:read`)

**Query Parameters:**
- `user_id` (optional): Filter by user ID.
- `limit` (optional): Default 20, max 100.

**Response (200 OK):**
```json
{
  "data": [
    {
      "id": "01907a5e-5df3-7b9a-9f44-3c1d4d2e7f3a",
      "user_id": "jane.doe",
      "profile_slug": "aion_std",
      "title": "Infrastructure Analysis Q2",
      "message_count": 5,
      "metadata": {"source": "web-portal"},
      "tags": ["audit", "q2"]
    }
  ]
}
```

### Get Conversation Details
`GET /v1/conversations/{conversation_id}` (Requires `conversations:read`)

**Response (200 OK):**
Identical to the structure of the creation response. Returns `404 Not Found` if the conversation does not belong to the API key's tenant or does not exist.

### Modify Conversation
`PATCH /v1/conversations/{conversation_id}` (Requires `conversations:write`)

Allows partial modification of the session information.

**Request Body (All fields optional):**
```json
{
  "title": "New Modified Title",
  "tags": ["audit", "q2", "updated"],
  "metadata": {"source": "web-portal", "edited": true}
}
```

### Delete/Archive Conversation
`DELETE /v1/conversations/{conversation_id}` (Requires `conversations:write`)

Performs a logical deletion (soft-delete by setting `archived_at`) and releases the allocated MCP Stdio connections in memory for that specific session.

---

### Retrieve Conversation Messages
`GET /v1/conversations/{conversation_id}/messages` (Requires `conversations:read`)

**Query Parameters:**
- `limit` (optional): Default 50, max 1000.
- `include_internal` (optional): Default `false`. If set to `true`, it also includes the technical messages of execution of the individual tools and the raw plans of the orchestrator, otherwise it only returns the messages visible to the end user (roles `user` and `assistant`).

**Response (200 OK):**
```json
{
  "data": [
    {
      "id": "01907a5e-5df3-7b9a-9f44-...",
      "role": "user",
      "content": "What is the status of the database?",
      "reasoning": null,
      "tool_name": null,
      "tool_call_id": null,
      "created_at": "2026-06-22T15:53:31Z"
    },
    {
      "id": "01907a5e-5df3-7b9a-9f45-...",
      "role": "assistant",
      "content": "The database responds correctly with latency < 2ms.",
      "reasoning": "I checked the logs using the sql_query tool...",
      "tool_name": null,
      "tool_call_id": null,
      "created_at": "2026-06-22T15:53:33Z"
    }
  ]
}
```

## Chat and Streaming

### Agent Pre-heating (Warming Cache)
`POST /v1/chat/prepare` (Requires Chat Auth)

Preemptively starts the MCP servers configured for the profile and warms the agent's cache. Useful to invoke when the client opens to avoid cold-start on the first message.

**Request Body:**
```json
{
  "conversation_id": "01907a5e-...",
  "profile": "aion_std",
  "user_id": "jane.doe",
  "agent_mode": "normal"
}
```

**Response (200 OK):**
```json
{
  "ok": true,
  "status": "warming",
  "conversation_id": "01907a5e-...",
  "mcp_errors": [],
  "has_errors": false
}
```

### Status
`GET /v1/chat/prepare/status` (Requires Chat Auth)

Performs polling of the loading status of MCPs.

**Query Parameters:**
- `conversation_id`: Conversation ID.
- `profile`: Profile name/slug.
- `user_id` (optional): User ID.

---

### Chat Stream (SSE)
`POST /v1/chat/stream` (Requires Chat Auth)

Sends a message to the agent and receives a response in real time via **Server-Sent Events (SSE)**.

**Request Body:**
```json
{
  "conversation_id": "01907a5e-...",
  "message": "What is the status of the Prometheus servers?",
  "attachments": [],
  "profile": "aion_std",
  "reasoning_effort": "medium",
  "thinking_enabled": true,
  "web_search_enabled": false,
  "sql_query_project": "default"
}
```

**Response Events (SSE):**

Each event in the flow is of type `event: message` and contains a JSON payload under the `data:` key.

1. **`turn_started`**: Indicates the start of the generation and provides the message IDs.
   ```json
   {"type": "turn_started", "user_message_id": "...", "assistant_message_id": "..."}
   ```
2. **`token`**: A text chunk of the agent's response.
   ```json
   {"type": "token", "content": "Analyzing status..."}
   ```
3. **`reasoning`**: Chunk containing the internal reasoning of the agent (if supported by the model).
   ```json
   {"type": "reasoning", "reasoning": "Searching for servers..."}
   ```
4. **`tool_event`**: Call or completion events of an MCP tool.
   ```json
   {"type": "tool_event", "event": {"name": "run_command", "status": "running", "input": "..."}}
   ```
5. **`artifact_start` / `artifact_content` / `artifact_end`**: Signals the creation or modification of an artifact file generated by the agent (e.g. code, charts, reports).
   ```json
   {"type": "artifact_end", "artifact": {"identifier": "analis_prom", "type": "text", "path": "workspace/report.md", "version": 1, "saved": true}}
   ```
6. **`final`**: Contains the final consolidated text.
   ```json
   {"type": "final", "text": "The Prometheus servers are active..."}
   ```

### Stop Streaming
`POST /v1/chat/stop` (Requires Chat Auth)

Sends a cancellation signal to immediately stop the generation of the response in progress.

**Query Parameters:**
- `conversation_id`: ID of the conversation to stop.

## File Management (Attachments)

### Upload File
`POST /v1/conversations/{conversation_id}/files` (Requires `files:write`)

Allows multipart/form-data sending of files to be analyzed.
Returns an array of attachments saved in the `uploads/` folder of the session:
```json
{
  "attachments": [
    {
      "relative_path": "uploads/01907a5e_log.txt",
      "original_relative_path": "uploads/log.txt",
      "name": "01907a5e_log.txt",
      "original_name": "log.txt",
      "size_bytes": 1024,
      "mime": "text/plain"
    }
  ]
}
```

### List Session Files
`GET /v1/conversations/{conversation_id}/files` (Requires `files:read`)

Returns the list of files present in the session.

### Download File
Downloads of files associated with the session do **not** go through the `/v1/conversations/...` endpoints but through the legacy chat download endpoint:

`GET /sessions/{session_id}/download` (Requires Chat Auth)

**Query Parameters:**
- `relative_path`: The relative path of the file (e.g. `uploads/01907a5e_log.txt` or `workspace/report.md`).

## Common Errors and Rate Limiting

### Rate Limiting
The endpoints are protected by rate limiting. In case the set thresholds are exceeded, the server returns `429 Too Many Requests`.
**Response Headers**:
- `X-RateLimit-Limit`: Maximum limit allowed in the time window.
- `X-RateLimit-Remaining`: Remaining requests in the current window.
- `X-RateLimit-Reset`: UTC timestamp of the counter reset.

### Status Codes Table

| Code | Meaning | Typical Cause / Resolution |
|---|---|---|
| `401` | Unauthorized | API Key or Bearer Token missing, expired or invalid. |
| `403` | Forbidden | Correct API Key but lacking the necessary scopes (e.g. writing on read-only endpoint), or user not associated with the SQL project. |
| `404` | Not Found | Conversation, file or project does not exist or does not belong to the tenant. |
| `429` | Too Many Requests | Exceeded the limit of allowed calls per minute. |
| `503` | Service Unavailable | The unified SQLite database or Redis are offline. If the error indicates `Unified DB disabled`, you need to remove `AION_UNIFIED_DB=0` from the configurations to enable the v1 APIs. |
